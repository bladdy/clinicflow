================================================================================
                    DENTALBOT AI - FUNCIONALIDADES DEL SISTEMA
================================================================================

Plataforma SaaS multi-tenant para clinicas dentales.
Stack: ASP.NET Core 9 + SQLite + React + TypeScript + Tailwind CSS
IA: Ollama + Qwen3:4B (modelo local)
WhatsApp: Evolution API v2 (Baileys)
Arquitectura: Clean Architecture (Domain -> Application -> Infrastructure -> Api)

================================================================================
1. AUTENTICACION Y GESTION DE USUARIOS
================================================================================

- Login con email y contrasena (JWT + refresh tokens)
- Registro de cuentas con validacion de contrasena (8+ chars, mayuscula, minuscula, numero)
- 4 roles: Administrador, Recepcion, Doctor, SoloLectura
- Refresh tokens con expiracion de 7 dias
- Cuenta desactivable (IsActive)
- Endpoints: POST /api/auth/login, /register, /refresh, /revoke

12. ASISTENTE DE IA (OLLAMA + QWEN3)
================================================================================

- Generar respuestas automaticas a mensajes de pacientes
  * Contexto: ultimos 20 mensajes de la conversacion
  * Herramientas declaradas: BuscarHorarios, CrearCita, ReagendarCita, CancelarCita,
    BuscarPaciente, ConsultarServicios, ConsultarFAQ, TransferirHumano
- Detectar intencion del mensaje:
  * AGENDAR_CITA, REAGENDAR_CITA, CANCELAR_CITA, CONSULTAR_HORARIOS,
    CONSULTAR_PRECIOS, CONSULTAR_SERVICIOS, PREGUNTA_FAQ, URGENCIA, SALUDO, OTRO
- Detectar urgencias dentales:
  * Dolor intenso, sangrado abundante, trauma, hinchazon severa, fiebre
  * Si es urgencia: envia mensaje de emergencia y avisa que un doctor se comunicara
- Configuracion por empresa:
  * URL de Ollama, Modelo, Max tokens, Temperatura
  * Prompt del sistema (personalizable)
  * Mensaje de bienvenida y de transferencia
  * Toggle para activar/desactivar IA
- Endpoints: GET/PUT /api/ai-settings

15. FLUJO COMPLETO DEL BOT (END-TO-END)
================================================================================

1. Paciente envia mensaje por WhatsApp
2. Evolution API recibe el mensaje y envia webhook
3. Sistema crea/busca conversacion abierta para ese telefono
4. Asocia paciente automaticamente si el telefono coincide
5. Guarda mensaje entrante en la BD
6. Verifica si la IA esta habilitada para esa empresa
7. Si NO esta habilitada -> envia mensaje de transferencia a humano
8. Si SI esta habilitada:
   a. Recupera los ultimos 20 mensajes de contexto
   b. Analiza si es urgencia dental
   c. Si ES urgencia -> envia mensaje de emergencia
   d. Si NO es urgencia -> envia a Ollama con herramientas disponibles
   e. Ollama genera respuesta considerando horarios, citas, servicios, FAQ
   f. Guarda respuesta del bot y la envia al paciente

   - Mes, Total de citas, Completadas, Ingresos

- Selector de rango de fechas (Desde/Hasta)
- 5 tabs en el frontend
- Endpoints: GET /api/reports/overview, /appointments-by-service,
  /appointments-by-doctor, /monthly-trend, /patient-retention

================================================================================
19. NOTIFICACIONES Y RECORDATORIOS AUTOMATICOS
================================================================================

- Recordatorio de cita (24 horas antes):
  * Busca citas programadas para manana con estado Scheduled
  * Envia WhatsApp al paciente con details de la cita
  * Ejecucion cada hora via BackgroundService

- Encuesta post-cita:
  * Busca citas completadas ayer
  * Envia WhatsApp: "Califique del 1 al 5 su experiencia"
  * Ejecucion cada hora via BackgroundService

- Confirmacion de cita:
  * Envia WhatsApp al paciente con details de la cita confirmada
  * Incluye doctor, servicio y ubicacion

- Delay de 1 segundo entre mensajes para evitar rate limiting

23. SERVICIOS DE INFRAESTRUCTURA
================================================================================

1. AIService                                      - Generar respuestas, detectar intencion, detectar urgencia via Ollama
2. AuthService                                     - Login, Register, RefreshToken, RevokeToken
3. JwtTokenService                                 - Generar/validar tokens JWT, refresh tokens
4. EvolutionService                                - Comunicacion HTTP completa con Evolution API v2
5. WhatsAppService                                 - Capa de abstraccion sobre EvolutionService
6. NotificationService                             - Enviar recordatorios, confirmaciones y encuestas via WhatsApp
7. AppointmentReminderBackgroundService            - BackgroundService: ejecuta cada hora recordatorios y encuestas

24. MIDDLEWARE Y FUNCIONALIDADES TRANSVERSALES
================================================================================

- ExceptionMiddleware: Mapea excepciones a HTTP (401, 400, 404, 500) con formato ApiResponse
- CORS: AllowAll (cualquier origen, metodo y header)
- Swagger: Habilitado en Development con soporte JWT Bearer
- Soft Delete: Todas las entidades soportan eliminacion logica
- SeedData: Crea 4 roles, 1 empresa, 1 sucursal, 1 admin (admin@dentalbot.com / Admin123!), 4 planes
